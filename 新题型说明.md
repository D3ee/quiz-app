# 新题型功能说明

## 概述

答题系统已扩展支持 7 种题型，除了原有的单选和多选题，新增了：

1. **判断题** (judge)
2. **填空题** (fill)
3. **简答题** (short)
4. **代码题** (code)
5. **排序题** (order)

## 题型详解

### 1. 判断题 (judge)

用户点击"正确"或"错误"按钮作答。

```typescript
{
  id: '1001-javascript',
  type: 'judge',
  category: 'javascript',
  question: 'JavaScript 中 `null == undefined` 返回 true',
  answer: true,  // 布尔值
  explanation: '在 JavaScript 中，使用 == 进行比较时，null 和 undefined 被认为是相等的。',
}
```

### 2. 填空题 (fill)

用户在输入框中填写答案，支持多个空格。

```typescript
{
  id: '1004-javascript',
  type: 'fill',
  category: 'javascript',
  question: '在 JavaScript 中，可以使用 `___` 关键字声明块级作用域变量，使用 `___` 声明常量。',
  answer: ['let', 'const'],  // 字符串数组
  blanks: 2,  // 空格数量
  explanation: 'let 用于声明块级作用域的变量，const 用于声明常量。',
}
```

**注意**：填空题答案比较时会忽略大小写和首尾空格。

### 3. 简答题 (short)

用户在文本框中输入简短答案。

```typescript
{
  id: '1006-javascript',
  type: 'short',
  category: 'javascript',
  question: '请简述 JavaScript 事件循环（Event Loop）的执行顺序。',
  answer: '执行同步代码 -> 执行微任务队列 -> 执行宏任务队列 -> 重复',  // 字符串
  explanation: 'JavaScript 事件循环的基本流程：1) 执行同步代码；2) 执行所有微任务...',
}
```

### 4. 代码题 (code)

用户在代码编辑器中输入代码，可提供代码模板。

```typescript
{
  id: '1008-javascript',
  type: 'code',
  category: 'javascript',
  question: '实现一个防抖函数 debounce，延迟 delay 毫秒后执行。',
  codeTemplate: `function debounce(fn, delay) {
  // 请在此处实现
}`,
  answer: `function debounce(fn, delay) {
  let timer = null
  return function(...args) {
    clearTimeout(timer)
    timer = setTimeout(() => fn.apply(this, args), delay)
  }
}`,
  explanation: '防抖函数的核心思想是：在事件触发后延迟执行...',
}
```

### 5. 排序题 (order)

用户通过拖拽将选项排列成正确顺序。

```typescript
{
  id: '1010-javascript',
  type: 'order',
  category: 'javascript',
  question: '将以下 JavaScript 代码执行顺序排列正确：',
  options: [
    'console.log(1)',
    'setTimeout(() => console.log(2), 0)',
    'Promise.resolve().then(() => console.log(3))',
    'console.log(4)',
  ],
  answer: [0, 3, 2, 1],  // 正确顺序的索引数组
  explanation: '执行顺序：同步代码先执行（1, 4），然后执行微任务 Promise（3）...',
}
```

## 技术实现

### 类型定义更新

`src/types/index.ts` 中的 `Question` 接口已更新：

```typescript
export interface Question {
  id: string
  type: 'single' | 'multiple' | 'judge' | 'fill' | 'short' | 'code' | 'order'
  category: 'javascript' | 'vue2' | 'vue3' | 'miniprogram'
  question: string
  options?: string[]  // 可选，单选/多选/排序题需要
  answer: number | number[] | boolean | string | string[]
  explanation: string
  blanks?: number  // 填空题：空格数量
  codeTemplate?: string  // 代码题：代码模板
}
```

### Store 更新

`src/stores/quiz.ts` 新增 `isAnswerCorrect` 方法，统一处理所有题型的答案判断逻辑。

### 组件更新

`src/components/QuestionCard.vue` 已完全重构，支持所有 7 种题型的渲染和交互。

## 示例题库

新增的示例题库文件：`src/data/advanced.ts`

包含 15 道示例题目，涵盖所有新题型。这些题目已自动合并到各个分类的题库中。

## 使用方式

1. **添加新题目**：在对应分类的题库文件中添加题目，或在 `advanced.ts` 中添加
2. **题型选择**：根据题目特点选择合适的题型
3. **答案格式**：确保 `answer` 字段的类型与题型匹配
4. **可选字段**：
   - 填空题需要设置 `blanks` 字段
   - 代码题可以提供 `codeTemplate`
   - 单选/多选/排序题需要 `options` 数组

## 答案判断规则

- **单选/多选**：精确匹配索引
- **判断题**：布尔值精确匹配
- **填空题**：忽略大小写和首尾空格
- **简答/代码题**：忽略首尾空格（可根据需求调整为模糊匹配）
- **排序题**：精确匹配顺序

## 扩展建议

1. **代码题评测**：可集成代码执行引擎，实现自动化测试
2. **简答题评分**：可使用 AI 进行语义相似度评分
3. **填空题增强**：支持正则表达式匹配
4. **排序题优化**：添加拖拽动画效果
5. **新题型**：可继续扩展匹配题、连线题等

## 注意事项

- 所有题目的 `id` 必须唯一
- `category` 必须是已定义的分类之一
- 确保 `answer` 类型与题型匹配，否则会导致判断错误
- 代码题和简答题的答案比较较为严格，建议提供明确的答案格式说明
